# Environment Variables — paa-slack-monitor

Set these on the Railway service. Local dev: copy `.env.example` to `.env`.

## Postgres
| Var | Required | Notes |
| --- | --- | --- |
| `DATABASE_URL` | yes | Railway-provided Postgres connection string. |

## Auth
| Var | Required | Notes |
| --- | --- | --- |
| `DASHBOARD_PASSWORD` | yes | Password for `/login`. Single-user app. |
| `SESSION_SECRET` | yes | Random 32+ byte hex (e.g. `openssl rand -hex 32`). Signs the session cookie + the OAuth state token. |
| `CRON_SECRET` | yes | Bearer token for `/api/cron/*`. Embed in the cron service's startCommand per the Railway cron-secret convention. |

## Slack App (api.slack.com/apps → A0B49GEKJMN)
| Var | Required | Notes |
| --- | --- | --- |
| `SLACK_CLIENT_ID` | yes | From "Basic Information" → App Credentials. |
| `SLACK_CLIENT_SECRET` | yes | From "Basic Information" → App Credentials. |
| `SLACK_SIGNING_SECRET` | optional | Needed only if/when we add Slack Events API webhooks (not used by the polling scanner). |
| `PUBLIC_BASE_URL` | yes | The public URL of the deployed dashboard (no trailing slash). MUST match the Slack app's OAuth Redirect URL exactly. Example: `https://slack.palmettoaiautomation.com`. |

## Anthropic
| Var | Required | Notes |
| --- | --- | --- |
| `ANTHROPIC_API_KEY` | yes | Used by Haiku (relevance scoring) and Sonnet (response drafting). |

## Tuning
| Var | Required | Notes |
| --- | --- | --- |
| `SLACK_EXCLUDE_CHANNELS` | optional | Comma-separated substring patterns matched against channel names (case-insensitive). Default: `jobs,hiring,classifieds,off-topic,random,memes`. |

## Slack OAuth — Scopes the app must request
The user-token install (`/api/slack/install`) requests these **User Token Scopes** (no bot scopes):
- `channels:history`
- `channels:read`
- `groups:history`
- `groups:read`
- `users:read`
- `users.profile:read`

Add these in Slack app settings → "OAuth & Permissions" → "User Token Scopes" before clicking Connect.

## Slack OAuth — Redirect URL
Add `${PUBLIC_BASE_URL}/api/slack/callback` to "OAuth & Permissions" → "Redirect URLs".

## What the `xapp-1-...` token is for
The app-level token in your screenshot is for Socket Mode / event authorizations only. It cannot read channel history. The polling scanner uses per-workspace **user tokens (`xoxp-...`)** generated by the OAuth install flow and stored in the `slack_workspaces` table.
